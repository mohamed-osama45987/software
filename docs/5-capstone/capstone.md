# Capstone Project Brief

## Project Brief

This is the capstone project that will allow you to gauge your understanding of the topics learned throughout the programme.

The project is categorized into two main difficulties: **Basic** and **Advanced**. Choose the difficulty that suits your level of comfort during the project timeframe.

The project duration is **two weeks**, and instructors will be available for any questions. There will be check-in sessions during the two weeks and a presentation at the end.

This project can be done **individually or in groups**. Collaborate with your coursemates via Disco or your own Whatsapp group. Use this time to ask questions and clarify concepts, fostering collaboration as in the software development industry.

**Note:** The project is optional and not graded. You may modify your project after the course completion.

**Benefits of doing the capstone project:**
- Apply concepts learned in the programme by solving a real-life problem.
- Create a developer portfolio with projects and exercises made throughout the programme.
- Add the project to your portfolio to showcase skills to prospective employers.

---

## Basic (Choose one only)

### 1. Frontend App (Web and/or Mobile) connecting to an existing API

#### a. Project Specifications

- Create a React app that connects to an API using AXIOS to fetch information.
- Apply concepts from the Frontend module:
  - Components, props, states
  - Hooks: `useState`, `useEffect`, `useContext`, `useReducer` (whichever needed)
  - Unidirectional data flow, CRUD operations
  - Axios and Routing
- Add your own styling. Optionally use libraries such as React Bootstrap and make your app responsive.
- Hosting on an external server is optional.

#### b. Project Ideas
- Carpark Availability
- Weather App
- Movie Database
- Hotel Review and Reservation

*You are not restricted to these ideas; feel free to use your own!*

#### c. API Sources
- [RapidAPI](https://rapidapi.com/collection/list-of-free-apis)
- [Free APIs on GitHub](https://github.com/public-apis/public-apis)
- [Mock APIs](https://mockapi.io/)

---

### 2. Backend API with Database on a Local Machine

#### a. Project Specifications
- Create a Spring Boot application that exposes RESTful APIs and connects to a database (H2/PostgreSQL).
- Apply concepts from the Backend module:
  - RESTful API Design (GET, POST, PUT, DELETE)
  - Spring Boot fundamentals (Spring MVC, Spring Data JPA)
  - Database connectivity (H2 for in-memory, PostgreSQL for persistence)
  - CRUD operations with proper request handling
  - DTOs and Model Mapping for data transfer
  - Exception Handling and Validation (`@ControllerAdvice`, `@Valid`)
  - Use Lombok to simplify code
  - Basic Security using Spring Security or JWT (optional)
  - Logging with SLF4J and Logback (optional)
- Implement testing (optional):
  - Basic Unit Testing with JUnit and Mockito

#### b. Project Ideas
- Product Management API
- Simple CRM (Customer Relationship Management) API
- Task Management API
- Employee Management API

*You are not restricted to these ideas; feel free to use your own!*

#### c. Tools and Technologies (use as many as you can)
- Spring Boot (Spring MVC, Spring Data JPA)
- Database: H2 (testing), PostgreSQL (persistence)
- Lombok (reduce boilerplate)
- Postman / ThunderClient for API testing
- JUnit, Mockito (for testing, if implemented)

## Advanced

### 1. Full Stack application (React FE with Java API Backend)

#### a. Project Specifications
- Create two applications: a React app and a Java API app, and have them communicate with AXIOS.
- Apply Frontend concepts:
  - Components, props, states
  - Hooks: `useState`, `useEffect`, `useContext`, `useReducer` (whichever needed)
  - Unidirectional data flow, CRUD operations
  - Axios and Routing
- Apply Backend concepts:
  - REST endpoints (CRUD)
  - SpringBoot components and Annotations
  - Databases
- Use your own styling: challenge yourself with libraries such as React Bootstrap and make your app responsive.
- Hosting is optional.

#### b. Project Ideas
- Customer Relationship Management System (CRM)
- Event Booking
- Hotel Review and Reservation

*You are not restricted to these ideas; feel free to use your own!*
